Artificial Intelligence for Stylus

Authors: Christian Hains, Roman Parks(and associates)

Abstract

Stylus is a program created in 2008 by Dr. Douglas Axe to simulate molecular evolution using Chinese characters as an analogy to protein structures. These are made up of strokes formed from a set of vectors analogous to the 20 amino acids. Stylus scores these drawn structures (scores ranging from 0 to 1) according to their resemblance to the ideal archetype. The original version requires the user to specify which character is being represented, and which vector stroke corresponds to each Chinese stroke. Last year we began a project to create an AI model that could recognize the origin of a mutated gene. The goals were for it to be able to find what character the mutated gene originally represented, and which stroke in the original chinese order was highlighted in red. The previous iteration of this project, an AI made to assist this scoring process, demonstrated the feasibility of a generalized scoring system that no longer has these constraints. The objective with this current project is to improve the scope and scale of that previous model, allowing Stylus to evaluate a much greater variety of genes. We are training our model, a dual branch CNN, with 540 million training images (an over 43 fold increase from the last model).

Pacing Improvements

    To ensure that gene creation and database upload could happen overnight, we added pacing functions to both the gene creation and the database pipeline. The gene creation had a buffer, such that if it created over a hardcoded amount of files in its output folder, it would pause until the number of files in that output folder once again dipped below that buffer. We made it so that the database upload system watched the inflow rate of the mutation’s output folder, and predicted the next second’s inflow rate accordingly. It then changed its attempted genes-shipped-per-second to match that rate. This worked wonders, making significantly more files than last year, with approximately one third of the compute for both mutation and database upload.

Multi-thread Improvements

Previously, our gene generation process was only using a single core of the 12 core CPU, mainly because a gene only needs one core. However, due to the massive amount of genes to generate, using more CPU cores would speed up the process significantly. We developed a program that generated independent sets of genes on each core, improving generation time efficiency by ~16 times.

Upload Improvements

After improving our generation speed, the next bottleneck was the upload speed to our database. This year, we improved the upload pipeline by first making the image-creation process much more efficient with the library pillow instead of matplotlib, which essentially meant we specified more details, in order to use a much more lightweight, less capable rendering engine. We also hosted the database server on a local computer, which ended up making things even faster, especially once we switched to doing our file transfers with a wired network connection, instead of the mesh network we started with. At the end of the day, this was 25.9 times faster than the matplotlib method, which was already faster than last year due to the personal hosting of the database.
